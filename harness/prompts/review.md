# 코드 리뷰 요청 프롬프트 (Code Review)

> 방금 작성/변경한 코드(또는 특정 파일/디프)를 리뷰받고 싶을 때 붙여넣으세요.

---

다음 변경사항을 **이 프로젝트의 규칙 기준**으로 리뷰해줘. 코드를 다시 쓰기 전에 먼저 **문제 목록과 우선순위**부터 제시할 것.

**리뷰 대상**: <파일 경로 / "방금 변경분" / `git diff`>

판단 기준 (근거로 삼을 문서):
- [CLAUDE.md](../../CLAUDE.md), [.claude/rules/code-style.md](../../.claude/rules/code-style.md), [.claude/rules/architecture.md](../../.claude/rules/architecture.md), [harness/checklist.md](../checklist.md)

다음 항목을 점검해줘:
1. **정확성/버그** — 엣지 케이스, null/에러 처리, 클라↔서버↔DB shape 정합성.
2. **레이어 위반** — 컴포넌트 직접 fetch, services 우회, 레이어 건너뜀.
3. **타입 안전성** — `any` 남용, `@maxt/shared` 미사용/중복 타입.
4. **서버 규칙** — `.js` 확장자, `verifyAuth` 누락, SQL 파라미터 바인딩.
5. **스타일** — 세미콜론/따옴표/네이밍, 한국어 문구 규칙.
6. **디자인** — HEX·px 하드코딩, `Cy*` 프리미티브 미활용.
7. **보안** — 시크릿 하드코딩, 입력 검증, 토큰 처리.
8. **범위/응집** — 요청 외 변경, 거대 파일, 중복.

출력 형식:
- 🔴 **반드시 수정** / 🟡 **개선 권장** / 🟢 **좋은 점** 으로 분류.
- 각 항목: `파일:라인` + 무엇이 문제인지 + 어떻게 고칠지(필요시 짧은 예시 코드).
- 마지막에 "전체 평가 한 줄"과 "다음 액션".

리뷰 후 내가 "고쳐줘"라고 하면 그때 수정한다. 그 전에는 진단만.
