# 자체 검증 체크리스트 (Self-Review Checklist)

> Claude는 코드 생성/수정을 마친 뒤 이 목록을 점검하고, 결과를 한두 줄로 요약해 보고한다.
> 어긴 항목이 있으면 고치거나, 못 고치면 이유를 명시한다.

## 1. 스타일 / 포맷
- [ ] 세미콜론 없음, 작은따옴표, 2-space 들여쓰기.
- [ ] 네이밍 규칙 준수 (컴포넌트 PascalCase, 훅 `useXxx`, 스토어 `useXxxStore`).
- [ ] 주석/식별자는 영어, **사용자 노출 문구·에러 메시지는 한국어**.

## 2. 타입
- [ ] 도메인 데이터에 `any` 미사용.
- [ ] 공유 타입은 `@maxt/shared`에 정의하고 import (중복 정의 없음).
- [ ] 타입 전용 import는 `import type` 사용.

## 3. 아키텍처 / 레이어
- [ ] 컴포넌트에서 직접 `fetch`/`axios` 호출하지 않음 → `services/` 경유.
- [ ] 인증 필요한 호출은 `http.ts`의 `api` 인스턴스 사용.
- [ ] (서버) 상대 import에 `.js` 확장자 포함.
- [ ] (서버) 인증 필요한 라우트에 `verifyAuth` 적용, `index.ts`에 라우터 등록.
- [ ] (서버) SQL은 파라미터 바인딩(`$1`)만 사용, 문자열 보간 없음.
- [ ] 새 파일이 [CLAUDE.md](../CLAUDE.md)의 위치 규칙에 맞는 폴더에 있음.

## 4. 디자인
- [ ] 색/여백/폰트 등은 `theme.ts` / `cy-*` / Tailwind 토큰 사용, HEX·px 하드코딩 없음.
- [ ] 재사용 가능한 UI는 `components/ui`의 `Cy*` 프리미티브 활용.
- [ ] 반응형 동작 확인(레이아웃이 좁은 화면에서 깨지지 않는지).

## 5. 보안 / 환경
- [ ] 시크릿·URL·키 하드코딩 없음 (`process.env` / `import.meta.env`).
- [ ] 토큰은 `localStorage('maxt-token')` 규약 유지.
- [ ] 사용자 입력 검증(빈 값/길이/권한) — 실패 시 한국어 에러 + early return.

## 6. 정합성
- [ ] 클라이언트 요청 shape ↔ 서버 핸들러 ↔ DB 컬럼이 일치.
- [ ] 변경이 기존 호출부를 깨뜨리지 않음 (참조 확인).
- [ ] 타입 체크 통과 (가능하면 `npm run build -w @maxt/shared` / client build로 확인).

## 7. 범위
- [ ] 요청 범위를 벗어난 변경 없음 ("겸사겸사" 리팩토링 금지).
- [ ] 커밋/푸시는 사용자가 요청했을 때만 수행.

---
**보고 형식 예시:**
> ✅ 스타일/타입/레이어 통과. ⚠️ 타입체크는 환경상 미실행 — 사용자가 `npm run build:client`로 확인 필요. 시크릿 하드코딩 없음.
