# 명령: 버그 수정 (fix-bug)

아래 빈칸을 채워서 Claude에게 전달하세요.

---

## 버그 내용
- **증상**: <무엇이 잘못 동작하는가>
- **재현 방법**: <어떤 화면/요청에서, 어떤 순서로>
- **기대 동작**: <원래 어떻게 되어야 하는가>
- **에러/로그**: <콘솔·네트워크·서버 로그 붙여넣기>
- **발생 범위**: <client / server / shared / DB 중 어디로 추정?>

## Claude가 따라야 할 절차
1. **추측하지 말고 좁혀라**: 관련 파일을 읽고 데이터 흐름(`UI → store → services → http` / `route → middleware → db`)을 따라가며 원인 후보를 찾는다.
2. **근본 원인 설명**: 수정 전에 "무엇이 왜 깨졌는지" 한두 문장으로 먼저 설명한다.
3. **최소 변경**: 증상만 가리는 패치가 아니라 원인을 고친다. 단, 범위는 버그에 한정 — 무관한 리팩토링 금지.
4. **회귀 방지**: 같은 실수가 다른 곳에도 있는지 확인하고 알려준다.
5. **검증**: 재현 시나리오 기준으로 어떻게 확인했는지(또는 사용자가 확인할 방법) 명시. [harness/checklist.md](../../harness/checklist.md) 점검.

## 자주 보는 함정 (먼저 의심해볼 것)
- 서버 ESM 상대 import에서 `.js` 확장자 누락.
- `@maxt/shared` 타입과 실제 DB 컬럼/응답 shape 불일치.
- JWT 토큰 누락/만료 → 401 후 자동 로그아웃 동작.
- SQL 파라미터 바인딩 누락, `null` 처리.
- zustand 상태 비동기 업데이트 순서, `isLoading`/`error` 초기화 누락.
- 클라이언트에서 직접 `fetch`를 써서 인증 헤더가 안 붙는 경우 → `api` 인스턴스로 교체.
